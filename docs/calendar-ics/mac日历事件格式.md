# 日历格式

## mac 日历创建的格式

```conf
BEGIN:VCALENDAR
VERSION:2.0
PRODID:-//PYVOBJECT//NONSGML Version 1//EN
X-WR-CALNAME;VALUE=TEXT:个人
BEGIN:VTIMEZONE
TZID:Asia/Shanghai
BEGIN:DAYLIGHT
DTSTART:20000101T000000
RRULE:FREQ=YEARLY;BYMONTH=1
TZNAME:GMT+8
TZOFFSETFROM:+0900
TZOFFSETTO:+0900
END:DAYLIGHT
END:VTIMEZONE
BEGIN:VEVENT
UID:99A367A7-02C7-4610-9985-C1CD8C9DE5BA
DTSTART;TZID=Asia/Shanghai:20211120T140000
DTEND;TZID=Asia/Shanghai:20211120T160000
CREATED:20211120T023933Z
DESCRIPTION:1. sql请求过程\n2. 事务\n3. acid\n4. sql优化
DTSTAMP:20211120T060816Z
LAST-MODIFIED:20211120T045048Z
SEQUENCE:2

TRANSP:OPAQUE
URL;VALUE=URI:

UID:9003784A-0ECA-421A-BDB4-71045B7EFD89

END:VEVENT
END:VCALENDAR
```

### `VEVENT` 事件格式分析

```js
BEGIN:VEVENT                                            // 事件解析开始
UID:99A367A7-02C7-4610-9985-C1CD8C9DE5BA                // UID
DTSTART;TZID=Asia/Shanghai:20211120T140000              // 开始时间
DTEND;TZID=Asia/Shanghai:20211120T160000                // 结束时间
CREATED:20211120T023933Z                                // 创建时间
DESCRIPTION:1. sql请求过程\n2. 事务\n3. acid\n4. sql优化   // 详情/备注
DTSTAMP:20211120T060816Z                                // 时间戳 ??
LAST-MODIFIED:20211120T045048Z                          // 最近的更新时间
SEQUENCE:2                                              // 顺序 ???
SUMMARY:mysql知识点巩固/重新梳理                           // 标题/总结的标题
TRANSP:OPAQUE                                           // 交通运输部 ?? 出行方式?
URL;VALUE=URI:                                          // 连接的URL

BEGIN:VALARM
```
